# 알로소(Alloso) 소재 제작 규칙

이 문서는 "무엇을 만들지"에 대한 브랜드 규칙만 담고 있다. 파이프라인이 "어떻게 동작하는지"(API 호출,
이미지 크롭 알고리즘, Figma 렌더링 방식 등)는 [ARCHITECTURE.md](ARCHITECTURE.md)를 참고할 것.

각 규칙 항목마다 "수정 위치"를 표시해뒀다 — 규칙을 바꾸고 싶을 때 어느 파일의 어느 부분을 고쳐야 하는지
바로 찾을 수 있도록.

---

## 1. 기본 출력 사이즈

모든 소재의 기본 출력 사이즈는 **1080 x 1080px** (정사각형).

- 수정 위치: `vision/rules.js` → `OUTPUT_SIZE`

## 2. 폰트

| 문자 종류 | 폰트 | 자간(letter spacing) |
|---|---|---|
| 국문 | Pretendard | -1.5% |
| 숫자 | Pretendard | -1.5% |
| 영문(A-Z, a-z) | Century Gothic | -2.5% |

- 한 문장 안에 국문/영문/숫자가 섞여 있으면 문자 단위로 폰트가 자동 분기된다 (예: "7.18(토) 오전 11시"에서
  "7", "18", "토", "오전", "11", "시"는 Pretendard, "(", ")" 등 기호는 국문 그룹으로 처리).
- **폰트 굵기(weight)**: 메인 타이틀은 Bold, 서브 텍스트는 Regular. Vision 분석 단계에서 시안을 보고 판단.
- Regular(얇은) 텍스트는 시안에서도 가늘고 심플한 인상을 유지해야 함.
- 지정 폰트가 Figma 파일에 설치되어 있지 않으면 자동으로 Inter로 대체된다.

- 수정 위치:
  - 어떤 문자가 어느 폰트로 갈지 판단 규칙(국문/영문 분기 기준) → `code.ts` 상단 "소재 제작 규칙" 블록의
    `KR_FAMILY`, `LATIN_FAMILY`, `KR_LETTER_SPACING_PERCENT`, `LATIN_LETTER_SPACING_PERCENT`, `isLatinChar`
  - Vision이 굵기를 판단하는 기준 문구 → `vision/rules.js` → `PRODUCTION_RULES`의 `[폰트]` 섹션

## 3. 텍스트 크기/정렬

- 메인 타이틀: 68px 이내 권장 (고정값 아님, 시안 비율을 보고 판단)
- 서브타이틀: 메인 타이틀의 약 60%
- 정렬(좌/중앙/우): 시안에 보이는 정렬을 그대로 따름

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[텍스트]` 섹션

## 4. "LIVE" 단어 처리 — 항상 로고 이미지로 자동 치환

문장 안에 "LIVE"라는 단어가 등장하면, 텍스트로 렌더링하지 않고 **저장된 LIVE 로고 PNG 이미지로 자동
치환**해서 삽입한다. (예: "가격인상 전 LIVE 단독혜택" → "가격인상 전 [LIVE 로고 이미지] 단독혜택")

- Vision 분석 단계는 그냥 "LIVE"라는 글자를 텍스트 그대로 포함시키기만 하면 되고, 실제 이미지 치환은
  Figma 플러그인이 렌더링할 때 자동으로 처리한다.
- 이 동작은 `liveBadge` 에셋(아래 6번 항목)이 스펙에 있어야 작동한다.
- 인라인 로고의 높이 = 폰트 크기 × 1.05, 로고와 텍스트 사이 여백 = 폰트 크기 × 0.15 (비율로 조정됨).

- 수정 위치:
  - 치환 크기/여백 비율 → `code.ts` 규칙 블록의 `LIVE_INLINE_LOGO_HEIGHT_RATIO`, `LIVE_INLINE_LOGO_GAP_RATIO`
  - Vision에게 "LIVE는 그대로 텍스트에 남겨라"라고 지시하는 문구 → `vision/rules.js` → `PRODUCTION_RULES`의
    `[텍스트]`, `[LIVE 뱃지]` 섹션

## 5. 로고 배치

브랜드 로고(예: `알로소 로고_화이트.png`)는 시안에서 크롭하지 않고, 별도로 제공된 실제 PNG 파일을
그대로 사용한다. Vision 분석은 로고가 들어갈 위치/크기만 판단한다.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[로고]` 섹션

## 6. LIVE 뱃지 (날짜 옆 단독 뱃지)

빨간 사각형 배경에 흰색 "LIVE" 글자가 있는 뱃지(예: 날짜 텍스트 옆에 붙는 뱃지)도 로고와 마찬가지로
별도 PNG 파일(`LIVE 로고.png`)을 그대로 사용한다. 시안에 이 뱃지가 안 보이면 생략.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[LIVE 뱃지]` 섹션

## 7. 텍스트 강조(highlights)

"LIVE" 외에 특정 단어만 다른 색으로 강조되어 있는 경우(예: 브랜드 포인트 컬러), 그 단어와 색을
개별 지정해서 부분 색상 적용이 가능하다.

- 수정 위치: `vision/analyze.js`의 `MATERIAL_SPEC_SCHEMA` → `texts.items.properties.highlights`

## 8. 가독성 처리 (텍스트 배경 패널, backdrop)

텍스트가 배경 사진 위에 있어 가독성이 떨어질 것으로 판단되면, 텍스트 뒤에 패널을 깐다.

- **gradient**: 텍스트 아래쪽에 옅게 깔리는 그라데이션 패널 (위: 투명 → 아래: 지정 색상)
- **blur**: 블러 처리된 반투명 패널 (기본 블러 강도 20px)
- 패널 색상은 반드시 그 텍스트 주변 배경 이미지의 어두운 영역에서 추출한 값을 사용 (임의의 검정색 금지)
- 이미 단색/그라데이션 배경 위라 가독성 문제가 없으면 생략

- 수정 위치:
  - 기본 블러 강도 → `code.ts` 규칙 블록의 `DEFAULT_BACKDROP_BLUR_RADIUS`
  - 판단 기준 문구 → `vision/rules.js` → `PRODUCTION_RULES`의 `[가독성 처리 - backdrop]` 섹션

## 9. 이미지 크롭 매칭

원본(크롭 안 된 고해상도 사진)에서, 광고주가 준 시안과 동일한 구도(제품이 보이는 각도/비율/여백)를
재현하는 정사각형 영역을 찾아 크롭한다. 원본이 시안보다 훨씬 넓은 화각일 수 있으므로, 시안 속 피사체와
동일한 피사체를 원본에서 찾아 정렬 기준으로 삼는다.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[이미지 크롭]` 섹션

## 10. 톤앤무드

깔끔하고 절제된 톤, 텍스트+이미지 위주 구성, 그리드형 정렬 선호, 과한 장식 지양.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[톤앤무드]` 섹션

## 브랜드 에셋 파일 관례

현재 작업 폴더(`C:\Users\gram\Documents\ai 소재 제작`)는 아래처럼 구성되어 있다:

```
ai 소재 제작/
├── 원본이미지/     ← 크롭 안 된 고해상도 원본 사진들
└── 요소/           ← 브랜드 에셋 (로고 PNG, LIVE 로고 PNG 등)
```

새 소재를 만들 때마다 필요한 것:
1. 원본 이미지 (원본이미지/ 폴더)
2. 광고주 시안 (레이아웃 레퍼런스 이미지, 보통 스크린샷)
3. `요소/` 폴더의 로고/LIVE 로고 PNG (보통 고정, 매번 새로 줄 필요 없음)
