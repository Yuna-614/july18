# 알로소(Alloso) 소재 제작 규칙

이 문서는 "무엇을 만들지"에 대한 브랜드 규칙만 담고 있다. 파이프라인이 "어떻게 동작하는지"(API 호출,
이미지 크롭 알고리즘, Figma 렌더링 방식 등)는 [ARCHITECTURE.md](ARCHITECTURE.md)를 참고할 것.

각 규칙 항목마다 "수정 위치"를 표시해뒀다 — 규칙을 바꾸고 싶을 때 어느 파일의 어느 부분을 고쳐야 하는지
바로 찾을 수 있도록.

---

## 0. 소재 카테고리

알로소 소재는 4개 카테고리로 나뉜다: **네이버기획전 / 29cm기획전 / 제품인지 / 별도기획전**.
아래 1~10번 규칙은 4개 카테고리 모두에 공통 적용되는 규칙(`COMMON_RULES`)이고, 카테고리마다 달라지는
톤/레이아웃/문구 패턴은 카테고리별 레퍼런스 소재를 학습해서 `CATEGORY_RULES`에 별도로 채워나간다.

- 수정 위치: `vision/rules.js` → `CATEGORIES`(카테고리 목록), `CATEGORY_RULES`(카테고리별 규칙)
- `vision/analyze.js`의 `analyze()` / `buildMaterialSpec()`가 `category` 인자를 받으면
  `getProductionRules(category)`가 공통 규칙 + 해당 카테고리 규칙을 합쳐서 Gemini에 전달한다.
  CLI 사용 시 마지막 인자(또는 `--args-file`의 `category` 필드)로 카테고리명을 넘기면 된다.

**참고**: `제품인지` 카테고리는 한 장짜리가 아니라 여러 장(표지 1장 + 컬러/디테일 컷 여러 장)으로 구성된
슬라이드 세트다. 표지와 이후 슬라이드는 스타일이 완전히 다르므로(표지=큰 제품명 타이포, 이후=미니멀 태그),
슬라이드마다 `analyze.js`를 따로 실행해서 프레임을 각각 만들어야 한다. 자세한 내용은
`CATEGORY_RULES[제품인지]` 참고. 정사각형 외 세로형(스토리) 버전도 함께 제작되는 카테고리지만 세로형
파이프라인 지원은 아직 없음(필요 시 나중에 추가).

## 1. 기본 출력 사이즈

모든 소재의 기본 출력 사이즈는 **1080 x 1080px** (정사각형).

- 수정 위치: `vision/rules.js` → `OUTPUT_SIZE`

## 2. 폰트

| 문자 종류 | 폰트 | 자간(letter spacing) |
|---|---|---|
| 국문 | Pretendard | -1.5% |
| 숫자 | Pretendard | -1.5% |
| 영문(A-Z, a-z) | Century Gothic | -2.5% |

- 한 문장 안에 국문/영문/숫자가 섞여 있으면 문자 단위로 폰트가 자동 분기된다 (예: "7.18(토) 오전 11시"에서
  "7", "18", "토", "오전", "11", "시"는 Pretendard, "(", ")" 등 기호는 국문 그룹으로 처리).
- **폰트 굵기(weight)**: 메인 타이틀은 Bold, 서브 텍스트는 Regular. Vision 분석 단계에서 시안을 보고 판단.
- Regular(얇은) 텍스트는 시안에서도 가늘고 심플한 인상을 유지해야 함.
- 지정 폰트가 Figma 파일에 설치되어 있지 않으면 자동으로 Inter로 대체된다.

- 수정 위치:
  - 어떤 문자가 어느 폰트로 갈지 판단 규칙(국문/영문 분기 기준) → `code.ts` 상단 "소재 제작 규칙" 블록의
    `KR_FAMILY`, `LATIN_FAMILY`, `KR_LETTER_SPACING_PERCENT`, `LATIN_LETTER_SPACING_PERCENT`, `isLatinChar`
  - Vision이 굵기를 판단하는 기준 문구 → `vision/rules.js` → `PRODUCTION_RULES`의 `[폰트]` 섹션

## 3. 텍스트 크기/정렬

- 메인 타이틀: 68px 이내 권장 (고정값 아님, 시안 비율을 보고 판단)
- 서브타이틀: 메인 타이틀의 약 60%
- 정렬(좌/중앙/우): 시안에 보이는 정렬을 그대로 따름

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[텍스트]` 섹션

## 4. "LIVE" 단어 처리 — 항상 로고 이미지로 자동 치환

문장 안에 "LIVE"라는 단어가 등장하면, 텍스트로 렌더링하지 않고 **저장된 LIVE 로고 PNG 이미지로 자동
치환**해서 삽입한다. (예: "가격인상 전 LIVE 단독혜택" → "가격인상 전 [LIVE 로고 이미지] 단독혜택")

- Vision 분석 단계는 그냥 "LIVE"라는 글자를 텍스트 그대로 포함시키기만 하면 되고, 실제 이미지 치환은
  Figma 플러그인이 렌더링할 때 자동으로 처리한다.
- 이 동작은 `liveBadge` 에셋(아래 6번 항목)이 스펙에 있어야 작동한다.
- 인라인 로고의 높이 = 폰트 크기 × 1.05, 로고와 텍스트 사이 여백 = 폰트 크기 × 0.15 (비율로 조정됨).

- 수정 위치:
  - 치환 크기/여백 비율 → `code.ts` 규칙 블록의 `LIVE_INLINE_LOGO_HEIGHT_RATIO`, `LIVE_INLINE_LOGO_GAP_RATIO`
  - Vision에게 "LIVE는 그대로 텍스트에 남겨라"라고 지시하는 문구 → `vision/rules.js` → `PRODUCTION_RULES`의
    `[텍스트]`, `[LIVE 뱃지]` 섹션

## 5. 로고 배치

브랜드 로고(예: `알로소 로고_화이트.png`)는 시안에서 크롭하지 않고, 별도로 제공된 실제 PNG 파일을
그대로 사용한다. Vision 분석은 로고가 들어갈 위치/크기만 판단한다.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[로고]` 섹션

## 6. LIVE 뱃지 (날짜 옆 단독 뱃지)

빨간 사각형 배경에 흰색 "LIVE" 글자가 있는 뱃지(예: 날짜 텍스트 옆에 붙는 뱃지)도 로고와 마찬가지로
별도 PNG 파일(`LIVE 로고.png`)을 그대로 사용한다. 시안에 이 뱃지가 안 보이면 생략.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[LIVE 뱃지]` 섹션

## 6-1. 프로모션 뱃지 (캡슐형, 할인율/이벤트명)

완전히 둥근 모서리(캡슐/알약 모양)에 할인율·이벤트명 등이 들어간 뱃지(예: `~29% + COUPON`, `브랜드데이`)도
로고/LIVE 뱃지와 마찬가지로 텍스트가 이미 포함된 완성본 PNG(예: `29cm기획전_요소.png`)를 그대로 사용한다.
`29cm기획전` 카테고리에서 자주 등장. 시안에 이런 뱃지가 안 보이면 생략.

- 스펙 필드: `badgePlacement`(분석 결과) → `badge`(최종 스펙, base64 포함)
- 수정 위치:
  - Vision에게 캡슐형 뱃지를 인식시키는 문구 → `vision/rules.js` → `COMMON_RULES`의 `[프로모션 뱃지]` 섹션
  - 스키마 정의 → `vision/analyze.js`의 `MATERIAL_SPEC_SCHEMA.properties.badgePlacement`
  - Figma 렌더링(로고/LIVE 뱃지와 동일한 `placeImageAsset` 재사용) → `code.ts`의 `buildMaterial()` 마지막 부분
  - CLI 사용 시 `node vision/analyze.js <원본> <시안> [로고] [LIVE뱃지] [출력경로] [카테고리] [프로모션뱃지PNG경로]`
    또는 `--args-file`의 `badgePath` 필드로 전달

## 7. 텍스트 강조(highlights)

"LIVE" 외에 특정 단어만 다른 색으로 강조되어 있는 경우(예: 브랜드 포인트 컬러), 그 단어와 색을
개별 지정해서 부분 색상 적용이 가능하다.

- 수정 위치: `vision/analyze.js`의 `MATERIAL_SPEC_SCHEMA` → `texts.items.properties.highlights`

## 8. 가독성 처리 (텍스트 배경 패널, backdrop)

텍스트가 배경 사진 위에 있어 가독성이 떨어질 것으로 판단되면, 텍스트 뒤에 패널을 깐다. 사진 배경뿐 아니라
흰 벽처럼 단조로운 배경 위에서도 시안에 그라데이션 패널이 보이면 채운다.

- **gradient**: 텍스트 뒤에 깔리는 그라데이션 패널
  - `angle`(도, 0~360): 그라데이션 진행 방향. 생략 시 기본값 270(수직, 아래쪽이 진하고 위로 갈수록 옅어짐).
    29cm기획전처럼 대각선 그라데이션(예: 27도)이 쓰이는 경우도 있음
  - `stops`: 정지점 2개. 생략 시 기본값(0%: 투명 → 100%: 지정 색상)으로 처음부터 끝까지 균일하게 페이드.
    일정 구간까지 solid로 유지되다 그 뒤로 페이드되는 경우(예: 45%까지 solid, 92%까지 페이드) 이걸로 표현
- **blur**: 블러 처리된 반투명 패널 (기본 블러 강도 20px)
- 패널 색상(`color`)은 두 기준 중 시안에 가까운 쪽을 사용: (1) 텍스트 주변 배경 이미지의 어두운 영역에서 추출한 값,
  또는 (2) 텍스트 색과 대비되는 색 (흰 텍스트 → 검정 계열, 검정 텍스트 → 흰색 계열). 임의의 검정색 사용은 금지
- 이미 단색/그라데이션 배경 위라 가독성 문제가 없으면 생략

- 수정 위치:
  - 기본 블러 강도 → `code.ts` 규칙 블록의 `DEFAULT_BACKDROP_BLUR_RADIUS`
  - 그라데이션 방향/정지점 렌더링 로직 → `code.ts`의 `createTextBackdrop()` (angle은 배경 그라데이션과 동일하게
    `gradientTransformFromAngle()`을 재사용)
  - 스키마 정의 → `vision/analyze.js`의 `MATERIAL_SPEC_SCHEMA.properties.texts.items.properties.backdrop`
  - 판단 기준 문구 → `vision/rules.js` → `COMMON_RULES`의 `[가독성 처리 - backdrop]` 섹션 (29cm기획전 관련
    보강은 `CATEGORY_RULES[29cm기획전]`에 있음)

## 9. 이미지 크롭 매칭

원본(크롭 안 된 고해상도 사진)에서 정사각형 영역을 찾아 크롭한다. 최우선 기준은 **제품이 잘리지 않고
중앙에 안정적으로 들어오는 것**이며, 시안의 구도(줌 정도, 여백 느낌)는 참고용일 뿐이다. 원본이 시안보다
훨씬 넓은 화각일 수 있어서 시안 구도를 억지로 재현하려 하면 제품이 한쪽으로 쏠리거나 잘릴 수 있으므로,
그런 경우 "제품이 안 잘리고 중앙에 오는 것"을 우선한다. 원본에서 시안 속 피사체와 동일한 피사체를 먼저
찾은 뒤 그 피사체를 중심으로 크롭 영역을 잡는다.

- 수정 위치: `vision/rules.js` → `COMMON_RULES`의 `[이미지 크롭]` 섹션

## 10. 톤앤무드

깔끔하고 절제된 톤, 텍스트+이미지 위주 구성, 그리드형 정렬 선호, 과한 장식 지양.

- 수정 위치: `vision/rules.js` → `PRODUCTION_RULES`의 `[톤앤무드]` 섹션

## 브랜드 에셋 파일 관례

현재 작업 폴더(`C:\Users\gram\Documents\ai 소재 제작`)는 아래처럼 구성되어 있다:

```
ai 소재 제작/
├── 원본이미지/     ← 크롭 안 된 고해상도 원본 사진들
├── 요소/           ← 브랜드 에셋 (로고 PNG, LIVE 로고 PNG, 카테고리별 프로모션 뱃지 PNG 등)
├── 네이버기획전/    ← 네이버기획전 카테고리 레퍼런스 소재
└── 29cm기획전/      ← 29cm기획전 카테고리 레퍼런스 소재
```

새 소재를 만들 때마다 필요한 것:
1. 원본 이미지 (원본이미지/ 폴더)
2. 광고주 시안 (레이아웃 레퍼런스 이미지, 보통 스크린샷)
3. `요소/` 폴더의 로고/LIVE 로고 PNG (보통 고정, 매번 새로 줄 필요 없음)
4. (`29cm기획전`처럼 캡슐형 프로모션 뱃지가 쓰이는 카테고리라면) 해당 뱃지 PNG — 뱃지 문구가 소재마다 다르면
   그때그때 새로 잘라서 `요소/`에 추가해야 함 (로고/LIVE 뱃지처럼 고정 재사용이 아님)
